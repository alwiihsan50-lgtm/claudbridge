# Permanent Deployment: Render + Supabase

This is the mature setup. It removes the temporary `trycloudflare.com` URL entirely.

## 1. Supabase

Create a dedicated Supabase project for CloudBridge.

Run:

```sql
-- server/migrations/001_cloudbridge_supabase.sql
```

The migration creates:

- `cloudbridge_devices`
- `cloudbridge_pairing_sessions`
- `cloudbridge_clipboard`
- `cloudbridge_files`
- private Storage bucket `cloudbridge-files`

RLS is enabled. No public table policies are added because the FastAPI server uses the service role key server-side.

## 2. Render

Deploy the `server/` folder as a Python web service.

Build command:

```bash
pip install -r requirements.txt
```

Start command:

```bash
uvicorn main:app --host 0.0.0.0 --port $PORT
```

Environment:

```env
CLOUD_BRIDGE_TOKEN=<random-admin-token>
CLOUD_BRIDGE_PUBLIC_URL=https://<render-app>.onrender.com
SUPABASE_URL=https://<project-ref>.supabase.co
SUPABASE_SERVICE_ROLE_KEY=<service-role-key>
SUPABASE_STORAGE_BUCKET=cloudbridge-files
CLOUD_BRIDGE_FILE_TTL_SECONDS=86400
```

## 3. Windows

Install dependencies once:

```powershell
cd windows_agent
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Set `windows_agent/.env`:

```env
CLOUD_BRIDGE_BASE_URL=https://<render-app>.onrender.com
CLOUD_BRIDGE_TOKEN=<same-admin-token-as-render>
CLOUD_BRIDGE_DEVICE_ID=windows-main
POLL_INTERVAL_MS=1500
DOWNLOAD_DIR=C:\Users\alwii\Downloads\CloudBridge
```

Run:

```powershell
.\start-windows-agent.ps1
```

Use the tray icon to create the iPhone pairing link.

## 4. iPhone

Open the pairing link generated by the Windows tray app.

After pairing:

1. Open the Render URL in Safari.
2. Tap Share.
3. Tap Add to Home Screen.
4. Use CloudBridge from the Home Screen.
